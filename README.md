# Конспект-бот

Telegram-бот, который превращает текст, голосовые сообщения и документы в короткий структурированный конспект.

## Какую проблему решает и для кого

Во время учёбы накапливается много материала, который нужно быстро просмотреть: длинные пересланные сообщения из учебных чатов, голосовые от преподавателя или одногруппников, лекции в виде текстовых файлов, статьи в PDF. Прочитать/прослушать всё целиком долго. Бот решает эту задачу для **школьников, студентов и абитуриентов**: присылаешь материал — получаешь краткую выжимку ключевых идей за секунды.

## Как запустить

1. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Скопировать `.env.example` в `.env` и заполнить `TELEGRAM_BOT_TOKEN` (получить у [@BotFather](https://t.me/BotFather)).
3. По желанию — указать `LLM_PROVIDER=openai` или `anthropic` и соответствующий API-ключ для более качественной суммаризации. Если оставить `LLM_PROVIDER=none`, бот будет работать полностью локально (алгоритм TextRank).
4. Для распознавания голосовых сообщений на компьютере должен быть установлен `ffmpeg`.
5. Запустить бота:
   ```bash
   python bot.py
   ```

## Входные данные и результат

**На входе:** одно из
- текстовое сообщение (в том числе пересланное);
- голосовое или аудиосообщение;
- файл `.txt`, `.docx` или `.pdf`.

**На выходе:** сообщение с кратким конспектом. Для аудио дополнительно присылается полная расшифровка речи.

## Пример работы

Пользователь пересылает боту длинное сообщение из учебного чата → бот отвечает.

В LLM-режиме (`LLM_PROVIDER=openai/anthropic`) конспект приходит по четырём разделам:
```
📝 Конспект:

**Основные идеи**
- ...конкретная мысль текста своими словами...
- ...

**Технические детали и практические нюансы**
- ...цифры, условия, требования из текста, если есть...

**Риски и ограничения**
- ...что в тексте названо проблемой или условием, при котором что-то не сработает...

**Вывод**
...главный практический итог в 1-2 предложениях...
```

В локальном режиме (`LLM_PROVIDER=none`) конспект проще — это несколько предложений, выбранных из исходного текста алгоритмом TextRank, без деления на разделы (подробнее — см. «Ограничения» ниже).

## Инструменты и библиотеки

- **python-telegram-bot** — работа с Telegram Bot API (асинхронные хендлеры).
- **python-docx**, **pdfplumber** — извлечение текста из `.docx` и `.pdf`.
- **faster-whisper** — офлайн-распознавание речи (голосовые/аудио → текст).
- **sumy (TextRank)** — локальная экстрактивная суммаризация без внешних API.
- **openai / anthropic** (опционально) — суммаризация через LLM для более связного, "человеческого" конспекта.

## Роль ИИ

ИИ использовался в двух ролях:
1. **В процессе разработки** — для обсуждения архитектуры проекта, написания и рефакторинга кода, поиска и объяснения ошибок, а также оформления этого README.
2. **В самом продукте (опционально)** — если включён режим `LLM_PROVIDER=openai/anthropic`, языковая модель используется как движок суммаризации: она получает исходный текст и промпт с инструкцией составить конспект. По умолчанию продукт работает без ИИ в рантайме — алгоритм TextRank суммаризирует локально.

## Ограничения

- PDF-сканы без текстового слоя не распознаются (OCR не реализован).
- Распознавание речи локальной моделью Whisper на слабом CPU может занимать заметное время для длинных аудио. При самом первом голосовом сообщении модель (~500 МБ) скачивается из интернета и кэшируется на диск — это разовая задержка при первом запуске, дальше распознавание идёт без повторной загрузки.
- Локальный режим суммаризации (TextRank) — экстрактивный: он выбирает существующие предложения из текста, а не пересказывает своими словами. Более качественный, "человеческий" пересказ с разбором рисков и деталей доступен только в LLM-режиме.
- Бот не хранит историю сообщений между запросами — каждое сообщение обрабатывается независимо.
- Видеосообщения-«кружочки» (video note) и группы файлов (альбомы) пока не поддерживаются — только одно вложение за раз из списка выше.

## Идеи развития

- Добавить OCR (например, Tesseract) для сканированных PDF и фото.
- Сохранять историю конспектов пользователя и присылать их по запросу.
- Добавить кнопки "сделать короче / подробнее" под ответом.
- Поддержка видеосообщений-«кружочков» (video note): извлечение аудиодорожки и распознавание тем же движком, что и для голосовых.
- Автоматическое определение языка исходного материала (сейчас распознавание речи "заточено" под русский).
