# Конспект-бот

Telegram-бот, который превращает текст, голосовые сообщения и документы в короткий структурированный конспект.

## Какую проблему решает и для кого

Во время учёбы накапливается много материала, который нужно быстро просмотреть: длинные пересланные сообщения из учебных чатов, голосовые от преподавателя или одногруппников, лекции в виде текстовых файлов, статьи в PDF. Прочитать/прослушать всё целиком долго. Бот решает эту задачу для **школьников, студентов и абитуриентов**: присылаешь материал — получаешь краткую выжимку ключевых идей за секунды.

## Как запустить

1. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Скопировать `.env.example` в `.env` и заполнить `TELEGRAM_BOT_TOKEN` (получить у [@BotFather](https://t.me/BotFather)).
3. По желанию — указать `LLM_PROVIDER=openai` или `anthropic` и соответствующий API-ключ для более качественной суммаризации. Если оставить `LLM_PROVIDER=none`, бот будет работать полностью локально (алгоритм TextRank).
   - Если используется не официальный API, а OpenAI/Anthropic-совместимый провайдер (прокси/агрегатор) — дополнительно укажи `OPENAI_BASE_URL` или `ANTHROPIC_BASE_URL` и точное название доступной там модели.
4. Для распознавания голосовых сообщений на компьютере должен быть установлен `ffmpeg`.
5. Запустить бота:
   ```bash
   python bot.py
   ```

## Входные данные и результат

**На входе:** одно из
- текстовое сообщение (в том числе пересланное);
- голосовое или аудиосообщение;
- файл `.txt`, `.docx` или `.pdf`.

**На выходе:** сообщение с конспектом — список ключевых мыслей связными предложениями, важные слова и цифры выделены жирным. Для длинных и многотемных материалов конспект дополнительно делится на смысловые блоки с подзаголовками — модель сама придумывает их формулировки под содержание конкретного текста, а не берёт из фиксированного шаблона. В конце — короткий общий вывод. Для аудио дополнительно присылается полная расшифровка речи.

**Команды бота:**
- `/start` — приветствие
- `/help` — подробная справка по форматам и ограничениям
- `/about` — о проекте, ссылка на код
- `/mode` — какой режим суммаризации сейчас активен (LLM или локальный)

## Пример работы

Пользователь присылает статью «Искусственный интеллект в образовании» → бот отвечает (в LLM-режиме):

```
📝 Конспект:

*Возможности ИИ в обучении*

- За последние 5 лет ИИ перестал быть фантазией и уже используется
  в школах и вузах: *ИИ-ассистенты* проверяют сочинения, адаптивные
  платформы подбирают индивидуальные траектории...

- ИИ помогает с автоматизацией рутины: проверкой тестов, отчётностью
  и расписаниями, что освобождает учителям до *30%* рабочего времени...

*Риски и ограничения*

- Эксперты предупреждают о потере критического мышления...
- Есть риски, связанные с данными и приватностью...

ИИ в образовании — это эволюционный шаг, полезный только при
сохранении человеческого контакта и творческой свободы.
```

Подзаголовки (*Возможности ИИ в обучении*, *Риски и ограничения*) — не фиксированный шаблон, а формулировки, которые модель подобрала специально под эту статью; для другого текста они будут другими. Короткие или однотемные тексты конспектируются без подзаголовков — просто список пунктов.

В локальном режиме (`LLM_PROVIDER=none`) конспект проще — это несколько предложений, выбранных из исходного текста алгоритмом TextRank, без подзаголовков и без выделений (подробнее — см. «Ограничения» ниже).

## Инструменты и библиотеки

- **python-telegram-bot** — работа с Telegram Bot API (асинхронные хендлеры).
- **python-docx**, **pdfplumber** — извлечение текста из `.docx` и `.pdf`.
- **faster-whisper** — офлайн-распознавание речи (голосовые/аудио → текст).
- **sumy (TextRank)** — локальная экстрактивная суммаризация без внешних API.
- **openai / anthropic** (опционально) — суммаризация через LLM для более связного, "человеческого" конспекта; поддерживается как официальный API, так и любой OpenAI/Anthropic-совместимый провайдер через настраиваемый `base_url`.

## Роль ИИ

ИИ использовался в двух ролях:
1. **В процессе разработки** — для обсуждения архитектуры проекта, написания и итеративного улучшения промпта суммаризации, рефакторинга кода, поиска и объяснения ошибок, а также оформления этого README.
2. **В самом продукте (опционально)** — если включён режим `LLM_PROVIDER=openai/anthropic`, языковая модель используется как движок суммаризации: она получает исходный текст и промпт с инструкцией составить конспект, сама решает формат (список или связные абзацы) и при необходимости — подзаголовки. По умолчанию продукт работает без ИИ в рантайме — алгоритм TextRank суммаризирует локально.

Промпт суммаризации дорабатывался итеративно на реальных тестах: от жёсткой фиксированной структуры (одинаковые разделы для любого текста) — к гибкому формату, который подстраивается под тип и объём конкретного материала.

## Ограничения

- PDF-сканы без текстового слоя не распознаются (OCR не реализован).
- Распознавание речи локальной моделью Whisper на слабом CPU может занимать заметное время для длинных аудио. При самом первом голосовом сообщении модель (~500 МБ) скачивается из интернета и кэшируется на диск — это разовая задержка при первом запуске, дальше распознавание идёт без повторной загрузки.
- Локальный режим суммаризации (TextRank) — экстрактивный: он выбирает существующие предложения из текста, а не пересказывает своими словами. Более качественный, "человеческий" пересказ с разбором рисков и деталей доступен только в LLM-режиме.
- Бот не хранит историю сообщений между запросами — каждое сообщение обрабатывается независимо.
- Видеосообщения-«кружочки» (video note) и группы файлов (альбомы) пока не поддерживаются — только одно вложение за раз из списка выше.
- Ограничение частоты — не чаще одного сообщения на пользователя в несколько секунд (настраивается через `RATE_LIMIT_SECONDS`), защита от спама и от лишних трат на LLM API.
- Документы и аудио ограничены по размеру (по умолчанию 20 МБ, `MAX_DOCUMENT_SIZE_MB` / `MAX_VOICE_SIZE_MB`) — проверяется до скачивания файла.
- Очень длинный текст, вставленный в поле ввода Telegram напрямую (не файлом), может быть автоматически разбит клиентом Telegram на несколько отдельных сообщений при отправке — бот обработает их как независимые тексты. Для длинных материалов рекомендуется присылать `.txt`-файл.

## Идеи развития

- Добавить OCR (например, Tesseract) для сканированных PDF и фото.
- Сохранять историю конспектов пользователя и присылать их по запросу.
- Добавить кнопки "сделать короче / подробнее" под ответом.
- Поддержка видеосообщений-«кружочков» (video note): извлечение аудиодорожки и распознавание тем же движком, что и для голосовых.
- Автоматическое определение языка исходного материала (сейчас распознавание речи "заточено" под русский).
- Вынести rate-limit из памяти процесса во внешнее хранилище (Redis), чтобы лимиты не сбрасывались при перезапуске бота.
